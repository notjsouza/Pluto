# Pluto — Your Smart Financial Advisor

[![Demo video link](https://img.youtube.com/vi/qrPbmtIxojU/0.jpg)](https://youtu.be/qrPbmtIxojU)

A personal finance application that connects to your bank accounts via Plaid, tracks spending, detects subscriptions, and delivers AI-powered financial insights.

---

## Architecture

```
┌─────────────────────────────────────────────────────┐
│                  Next.js 15 Frontend                │
│              (React 18, Turbopack, App Router)      │
└────────────┬──────────────┬──────────────┬──────────┘
             │              │              │
             ▼              ▼              ▼
     ┌───────────┐  ┌───────────┐  ┌───────────┐
     │  OAuth    │  │  Plaid    │  │    AI     │
     │  Handler  │  │  Handler  │  │  Handler  │
     │  Lambda   │  │  Lambda   │  │  Lambda   │
     └─────┬─────┘  └─────┬─────┘  └─────┬─────┘
           │              │              │
           ▼              ▼              ▼
     ┌───────────┐  ┌───────────┐  ┌───────────┐
     │  Cognito  │  │  Plaid    │  │  OpenAI   │
     │ User Pool │  │    API    │  │    API    │
     └───────────┘  └───────────┘  └───────────┘

```

### Backend — Java 21 Lambda Functions

Each function is a self-contained Maven module compiled to a fat JAR and deployed to AWS Lambda with a public Function URL (CORS handled in Java).

| Function | Handler class | Endpoints |
|---|---|---|
| `oauth-handler` | `com.pluto.oauth.handler.OauthHandler` | `GET /auth/google`, `GET /auth/google/callback`, `GET /debug` |
| `plaid-handler` | `com.pluto.plaid.handler.PlaidHandler` | `POST /create-link-token`, `POST /exchange-public-token`, `POST /accounts`, `POST /transactions` |
| `ai-handler` | `com.pluto.ai.handler.AiHandler` | `GET /health`, `GET /test`, `POST /analyze`, `POST /ask` |

A shared `common` module (`com.pluto.lambda.common.LambdaUtils`) provides CORS headers, response builders, and JSON parsing used by all three handlers.

### Frontend

- **Framework:** Next.js 15
- **Auth UI:** `@aws-amplify/ui-react`
- **Bank connection:** Plaid Link via `react-plaid-link`
- **AI insights:** rule-based + optional OpenAI GPT-4o via `ai-handler`

## Local Setup

### 1. Clone and install dependencies

```bash
git clone https://github.com/notjsouza/Pluto.git
cd Pluto
npm install
```

### 2. Configure environment variables

```bash
cp .env.example .env.local
```

Fill in `.env.local`:

```env
# Google OAuth
GOOGLE_CLIENT_ID=
GOOGLE_CLIENT_SECRET=

# Plaid
PLAID_CLIENT_ID=
PLAID_SECRET=
PLAID_ENV=sandbox

# OpenAI
OPENAI_API_KEY=

# Lambda Function URLs (populated after first deploy)
NEXT_PUBLIC_OAUTH_HANDLER_URL=
NEXT_PUBLIC_PLAID_HANDLER_URL=
NEXT_PUBLIC_AI_HANDLER_URL=
```

### 3. Build the Java Lambda functions

```bash
cd amplify/functions
mvn clean package -DskipTests
cd ../..
```

### 4. Deploy the Amplify sandbox + run the dev server

```bash
# Deploys backend (Lambda functions + Cognito) and starts the sandbox watcher
npm run bootstrap

# In a separate terminal
npm run dev
```

`bootstrap.sh` validates your `.env.local`, deploys the sandbox via `ampx sandbox`, and generates `amplify_outputs.json` with the live Lambda URLs and Cognito config.

Open [http://localhost:3000](http://localhost:3000).

## Deployment

Pushes to `master` trigger the Amplify CI/CD pipeline defined in [amplify.yml](amplify.yml), which:

1. Installs JDK 21 (Amazon Corretto) if not present
2. Runs `mvn clean package -DskipTests` to build all Lambda JARs
3. Runs `ampx pipeline-deploy` to update Lambda functions + Cognito via CDK
4. Builds and deploys the Next.js frontend

After each deploy, the Lambda Function URLs are emitted as Amplify outputs (`custom.oauthHandlerUrl`, `custom.plaidHandlerUrl`, `custom.aiHandlerUrl`) and must be set as environment variables in the Amplify Console for the frontend build.
